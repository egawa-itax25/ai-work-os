# 全画面共通の同期状態・ログイン状態表示

## 背景

AI Work OS は localStorage と Supabase を併用して、PC・スマホ・別PCで同じワークスペースを扱う設計になっている。
ただしユーザーからは、現在ログインしているか、ローカル保存だけなのか、クラウド同期済みなのか、保存中か、同期エラーかが分かりにくい。

## 決定

全画面共通で同期ステータスを表示する。
既存の `loadSyncedState` / `saveSyncedState` / Supabase Auth を置き換えず、同期状態を通知・表示するレイヤーを追加する。

## UI

- 未ログイン: この端末だけに保存中
- ログイン中: メールアドレスを表示
- 保存中: 保存中...
- 保存成功: クラウド同期済み
- 保存失敗: 同期エラー
- ローカル更新: ローカル変更あり

ステータスをクリックすると詳細パネルを開き、ログイン状態、保存先、最終同期時刻、手動同期、再試行、ログイン・ログアウト導線を表示する。

## データ保護

ログインした瞬間に localStorage のプロジェクトやタスクを消してはいけない。
同期時はローカルにしかないデータとクラウドにしかないデータを保持し、同じIDのデータは既存の更新時刻・同期メタ情報に従って新しい側を優先する。
上書き前にはローカルバックアップを残し、同期失敗時も localStorage を破棄しない。

## 実装方針

- `src/lib/sync-status.ts` に同期状態の小さなクライアントストアを追加する。
- `src/lib/synced-storage.ts` は既存の保存・読込処理を維持し、状態イベントを publish する。
- `src/components/sync-status-indicator.tsx` を追加し、`WorkspaceShell` から全画面に表示する。
- Supabase未設定時や未ログイン時も、アプリはローカル保存で継続動作する。

## 完了条件

- 全画面でログイン状態と同期状態が分かる。
- 詳細パネルでアカウント、保存先、最終同期時刻を確認できる。
- 手動同期と再試行ができる。
- 既存データを消さない。
- スマホでも状態が確認しやすい。

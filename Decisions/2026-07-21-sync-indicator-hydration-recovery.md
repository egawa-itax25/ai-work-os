# 同期表示の初期描画を安定化する

## 背景

全画面共通の同期表示で、サーバー描画時は Supabase 未設定、ブラウザの初期描画時は設定済みと判定されていた。この差により React のハイドレーション不一致が発生し、一部端末では画面全体がクライアント例外になった。

## 決定

- サーバーとブラウザの初期描画は、共通の未確定状態として描画する。
- ブラウザへのマウント完了後に Supabase の設定有無と認証状態を反映する。
- localStorage、クラウドデータ、統合処理には変更を加えない。
- lint、型検査、本番ビルドに加えて公開環境のブラウザログを確認する。

## 理由

同期とデータ保護の仕組みを維持したまま、端末や保存状態によって初期 HTML が変わる問題だけを解消できるため。
